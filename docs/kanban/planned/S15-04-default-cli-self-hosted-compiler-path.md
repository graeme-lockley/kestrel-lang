# Default CLI Self-Hosted Compiler Path

## Sequence: S15-04
## Tier: Optional
## Former ID: (none)

## Epic

- Epic: [E15 Bootstrap JAR Self-Hosting Handoff](../epics/unplanned/E15-bootstrap-jar-self-hosting-handoff.md)
- Companion stories: S15-01, S15-02, S15-03, S15-05

## Summary

Switch default `kestrel build`, `kestrel run`, and `kestrel test` compilation behavior to use self-hosted compiler classes generated by bootstrap, ensuring normal command execution no longer depends on bootstrap JAR or TypeScript compiler invocation in the steady state.

## Current State

`scripts/kestrel` currently compiles through `node compiler/dist/cli.js` and uses self-hosted CLI forwarding only as a scaffold. Even with bootstrap artifacts present, normal build/run/test paths still rely on TypeScript compiler execution.

## Relationship to other stories

- **Depends on**: S15-03 mode/provenance state.
- **Depends on**: S15-02 bootstrap command generating self-hosted compiler classes.
- **Blocks**: S15-05 CI handoff enforcement.

## Goals

1. Route `build`, `run`, and `test` compile operations through self-hosted compiler classes by default when bootstrap state is valid.
2. Ensure bootstrap JAR is not used as a compile fallback for normal command execution.
3. Preserve current feature parity for compilation flags (`--clean`, `--refresh`, `--allow-http`, status output) when switching backend.

## Acceptance Criteria

- After successful `./kestrel bootstrap`, default `./kestrel build hello.ks` performs compilation without invoking `node compiler/dist/cli.js` in normal mode.
- `./kestrel run` and `./kestrel test` use the same default self-hosted compiler path.
- Normal command flow does not invoke bootstrap JAR compilation after bootstrap succeeds.

## Risks / Notes

- Command parity regressions can hide in less common flags (`--status`, URL import modes, cache-clean behavior).
- Recursive CLI dispatch loops are possible if self-hosted command forwarding is not bounded clearly.
- Need deterministic behavior across macOS/Linux shells and CI runner environments.

## Impact analysis

| Area | Change |
|------|--------|
| CLI dispatch | Replace scaffolded TS-backed self-host forwarding with direct JVM execution of bootstrap-generated self-hosted CLI classes when mode is `self-hosted`. |
| Default command path | `run`, `dis`, `build`, and `test` default to self-hosted compiler path after successful bootstrap state validation. |
| Fallback behavior | Keep TypeScript path only for `bootstrap-required` mode; do not execute bootstrap JAR in normal command flow. |
| State gating | Use S15-03 state/provenance readers to deterministically choose active command backend. |
| Docs/specs | Document default mode-switch behavior and explicit fallback conditions. |
