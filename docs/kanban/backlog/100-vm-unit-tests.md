# VM Unit and Integration Tests

## Priority: 100 (Medium)

## Summary

The VM has essentially no meaningful tests. There are only 3 trivial Zig tests (a placeholder, a value size check, and a minimal .kbc load). No tests exist for any opcode execution, GC correctness, closure handling, record operations, ADT matching, exception handling, or multi-module loading. This makes it difficult to refactor or fix VM bugs with confidence.

## Current State

- `main.zig`: `test "vm placeholder" { expect(true); }`
- `value.zig`: `test "value size" { expect(@sizeOf(Value) == 8); }`
- `load.zig`: `test "load minimal kbc"` -- loads `test/fixtures/empty.kbc`, checks first byte is 0x11 (RET).
- `test/unit/` and `test/integration/` are empty (`.gitkeep` only).
- `test/fixtures/` has only `empty.kbc`.

## Acceptance Criteria

- [ ] **Value tests**: Tag encoding/decoding for int, bool, unit, char, ptr, fn_ref. Boundary values (max/min 61-bit int).
- [ ] **Arithmetic tests**: ADD, SUB, MUL, DIV, MOD, POW with various int values. Edge cases (zero, negative, large).
- [ ] **Comparison tests**: EQ, NE, LT, LE, GT, GE for ints and bools.
- [ ] **Control flow tests**: JUMP, JUMP_IF_FALSE with hand-crafted code buffers.
- [ ] **Record tests**: ALLOC_RECORD, GET_FIELD, SET_FIELD, SPREAD.
- [ ] **ADT tests**: CONSTRUCT, MATCH dispatch by constructor tag.
- [ ] **Closure tests**: LOAD_FN, MAKE_CLOSURE, CALL_INDIRECT for both fn_ref and closure.
- [ ] **Exception tests**: TRY, THROW, END_TRY -- handler invocation, normal completion.
- [ ] **GC tests**: Allocate many objects, trigger GC, verify no use-after-free. Mark/sweep correctness.
- [ ] **Loader tests**: Additional .kbc fixtures (multi-function, with ADT table, with shape table).
- [ ] Fixtures can be hand-crafted byte arrays or generated by the compiler.
