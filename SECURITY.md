# Security

## Reporting a vulnerability

If you believe you have found a security vulnerability in Kestrel, please report it responsibly so it can be addressed before wider disclosure.

**Preferred:** if the project is on GitHub, use **Security → Report a vulnerability** to open a *private* advisory. Otherwise use whatever private channel the maintainers publish for this repository.

**Alternative:** contact the maintainers privately. Do not open a public GitHub issue for undisclosed security problems.

Please include:

- A short description of the issue and its impact
- Steps to reproduce, or a proof-of-concept, if possible
- Affected versions or commit, if known

We will treat reports seriously and aim to respond in a reasonable timeframe. Thank you for helping keep users safe.

## Scope

This policy applies to the Kestrel compiler, VM, JVM runtime, CLI scripts, and official tooling in this repository. It does not cover third-party dependencies or downstream projects unless explicitly maintained here.
